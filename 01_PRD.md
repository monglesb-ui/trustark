# 01. PRD — 부동산 RAG 의사결정 코파일럿

## 1. 제품명

가칭: **집판단**

## 2. 제품 목적

부동산 계약 전 사용자가 직접 확인해야 하는 실거래가, 전월세 시세, 지역 통계, 건축물 정보, 지도 기반 입지 정보를 AI가 통합 분석하여 계약 전 위험 신호와 다음 액션을 제공한다.

## 3. 핵심 사용자

### 1차 사용자

- 전세·월세 계약을 앞둔 일반 사용자
- 매매를 고민하는 실수요자

### 2차 사용자

- 공인중개사
- 부동산 컨설턴트
- 임대관리 담당자
- 투자자

## 4. 사용자 문제

사용자는 부동산 계약 전에 다음 정보를 여러 사이트에서 직접 확인해야 한다.

- 주변 실거래가
- 전월세 보증금 수준
- 지역 가격 흐름
- 건축물 정보
- 주변 교통, 학교, 생활 편의시설
- 권리관계와 위험 신호

문제는 정보가 흩어져 있고, 일반 사용자가 이 정보를 해석하기 어렵다는 점이다.

## 5. MVP 목표

DLthon 기간 내 구현 목표는 다음으로 제한한다.

> 사용자가 주소와 계약 조건을 입력하면, mock data 또는 일부 API 데이터를 기반으로 리스크 리포트를 생성하는 웹 데모를 완성한다.

## 6. MVP 핵심 기능

| 우선순위 | 기능 | 설명 |
|---:|---|---|
| P0 | 주소/계약 조건 입력 | 주소, 계약 유형, 보증금, 월세, 매매가 입력 |
| P0 | 좌표 변환 | 네이버 Geocoding API 또는 mock geocoding으로 좌표 획득 |
| P0 | 실거래가 비교 | 주변 유사 거래 mock/API 데이터와 비교 |
| P0 | 전세가율/가격 위험 계산 | 보증금 또는 매매가 기준 간단한 위험 룰 적용 |
| P0 | 리스크 리포트 생성 | 위험도, 근거, 다음 액션 출력 |
| P1 | 지도 표시 | 대상 주소를 지도에 표시 |
| P1 | 주변 요소 표시 | 역, 학교, 실거래 비교 지점 표시 |
| P1 | RAG 근거 검색 | 정책/체크리스트/위험 기준 문서를 검색해 답변에 반영 |
| P2 | PDF 다운로드 | 리포트 PDF 저장 |
| P2 | 전문가 연결 CTA | 법무사/중개사 상담 연결 버튼 mock |

## 7. 사용자 플로우

1. 사용자가 웹 페이지 접속
2. 계약 유형 선택: 전세/월세/매매
3. 주소 입력
4. 보증금/월세/매매가 입력
5. 분석 시작 클릭
6. 시스템이 주소를 좌표로 변환
7. 실거래가/지역/건축물/mock 데이터를 조회
8. Risk Agent가 위험 점수를 계산
9. RAG가 계약 전 체크리스트 근거를 검색
10. Report Agent가 최종 리포트 생성
11. 지도와 리포트를 함께 표시

## 8. 리포트 출력 형식

### 리포트 구성

- 종합 위험도: 낮음 / 주의 / 검토 필요 / 위험
- 핵심 근거 3~5개
- 주변 시세 비교
- 지역 정보 요약
- 지도 기반 입지 정보
- 추가 확인 필요 서류
- 다음 액션 체크리스트

### 예시 출력

```text
종합 위험도: 주의

핵심 근거
1. 입력 보증금이 주변 유사 전세 거래 평균보다 높습니다.
2. 최근 해당 지역 거래량이 감소 추세입니다.
3. 건축물대장과 등기부등본 추가 확인이 필요합니다.

다음 액션
- 등기부등본 확인
- 건축물대장 확인
- 보증보험 가입 가능 여부 확인
- 특약 문구 검토
```

## 9. 안전한 표현 원칙

부동산 계약 판단은 고위험 의사결정이므로 AI는 단정적으로 말하지 않는다.

### 금지 표현

- “계약해도 됩니다.”
- “절대 안전합니다.”
- “사기입니다.”
- “투자해도 됩니다.”

### 권장 표현

- “현재 데이터 기준으로는 추가 확인이 필요합니다.”
- “위험 신호가 일부 확인됩니다.”
- “등기부등본 확인 전에는 최종 판단이 어렵습니다.”
- “전문가 검토를 권장합니다.”

## 10. 성공 기준

DLthon 발표 기준 성공 기준은 다음과 같다.

- 주소와 계약 조건을 입력할 수 있다.
- 분석 버튼을 누르면 리포트가 생성된다.
- 위험도와 근거가 표시된다.
- 지도에 대상 매물 위치가 표시된다.
- RAG/Agent 흐름이 발표에서 설명 가능하다.
- mock data 기반이라도 전체 사용 흐름이 끊기지 않는다.
