## 🎯 사용자 진짜 도움 기준 — 고도화 순서 7개

### 1️⃣ Competition Density 매장 수 정확화 (1순위, ★★★★★)

**왜 1순위**: 사용자가 가장 답답해하는 부분. "이 위치에 카페 몇 개?"는 창업 검토의 핵심.

**작업**:
- KATEC 변환 푸시 효과 미반영 원인 추적 (배포 안 됐는지 / mapx 응답에 없는지)
- Naver Local 쿼리 6 → 12개로 확장 (카페+카페 디저트+카페 베이커리+카페 커피)
- vworld geocode 병렬 호출 (5건 → 30건 가능)

**예상 효과**: 1건 → 8~15건 잡힘.
**예상 시간**: 1시간.

### 2️⃣ 상권분석 데이터 신뢰도 검증 (2순위, ★★★★★)

**왜 2순위**: 잘못된 수치(피크 00-06시, 수요일)는 데모에서 즉시 의심받음.

**작업**:
- 응답 raw 필드 1행을 trace에 노출 → 진짜 필드명 식별
- 자치구 매칭 로직 보강 (현재 ADSTRD_CD prefix 매칭이 부정확)
- "점포 19개" → 자치구 전체 평균이 아닌 상권당 평균이라 혼란. 명시 추가

**예상 효과**: 피크 17-21시 / 금토일 같은 현실적 수치.
**예상 시간**: 1시간.

### 3️⃣ 카드 위계 재정렬 + 빈 카드 숨김 (3순위, ★★★★)

**왜 3순위**: 정보 흐름이 명확해야 발표 시 흐름 좋음.

**제안 순서**:
```
1. Decision Card (결론)
2. 검토 위치 지도 (입지 확인)
3. 상권 진단 (시장 데이터)
4. 동종업종 + 건축물대장 (직접 평가)
5. 학교 정화구역 (있는 경우만)
6. 법령 RAG (근거)
7. 동네 분위기 (보조)
8. Agent 진단 패널 (개발자 모드)
```

**예상 시간**: 30분.

### 4️⃣ NEIS 학교 재시도 + 좌표 거리 (4순위, ★★★)

**왜 4순위**: NEIS 500은 일시적. 정상이면 "200m 안 학교 N개" 표시 가능.

**작업**:
- NEIS HTTP 500 시 재시도 (1회)
- 학교 주소 → 좌표 → 사용자 좌표 거리 계산
- 200m 절대보호구역 / 50m 상대보호구역 정확 표시

**예상 시간**: 30분 + NEIS 회복 시점.

### 5️⃣ Decision Card 액션 구체화 (5순위, ★★★)

**왜 5순위**: 결론은 명확한데 "즉시 할 일"이 일반론적.

**개선**:
- "강남구청 위생과 방문" → "강남구청 위생과 02-3423-XXXX 전화" (자치구별 연락처)
- "휴게음식점 영업신고서" → 링크: 식약처 식품안전나라
- "임대인 동의서" → 표준양식 다운로드 링크

**예상 시간**: 1시간.

### 6️⃣ 상권 진단 + 동네 분위기 통합 (6순위, ★★)

**왜 6순위**: 두 카드가 별도라 정보 분산.

**개선**: 상권 진단 카드 안에 Naver 뉴스 3건을 "최근 이슈" 섹션으로.

**예상 시간**: 1시간.

### 7️⃣ 카카오 재신청 / 다른 데이터 발급 (7순위, ★)

**왜 7순위**: 외부 의존 + 시간 걸림. 발표 후 가능.

**작업**:
- 카카오 재신청 (회사명/카테고리 보강)
- 공공데이터포털 "상가(상권)정보 API" 별도 발급
