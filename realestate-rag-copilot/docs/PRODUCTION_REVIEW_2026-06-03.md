# Production 분석 결과 평가 (2026-06-03)

대상 화면: `https://trust-ark.vercel.app` 분석 결과 페이지
사용 입력: 전세 / 서울 마포구 성산동 / 보증금 약 2.5억 / 빌라

## 잘된 점

1. **Mock 누수 차단 효과 확인** — 데이터 출처 상태 카드 7개가 명시적으로 상태(✓/실패/수동)를 노출. 직전 리팩터링(mock fallback skeleton 전환)이 production에서 의도대로 동작.
2. **실제 외부 API 연동 성공** — VWorld 지오코딩, data.go.kr 실거래가(전월세 110M / 매매 463M / 표본 14건), 건축물대장(사용승인일·주용도·층수)이 실데이터로 채워짐.
3. **등기부등본의 명시적 분리** — 자동 분석에서 권리관계 원문 호출을 생략하고 "수동 분석"으로 표시. 수수료·법적 안전 측면에서 옳은 분리.
4. **단정 표현 회피** — "검토 필요", "확인 권장" 톤 유지. Validation Agent가 작동.
5. **위험도 산정 근거 명시** — 60점 옆에 표본 수·전세가율 등 메타가 함께 보임.
6. **RAG 근거 분리 노출** — 위험 신호와 별도 카드로 RAG 체크리스트 표시.

## 부족한 점

1. **위험도 ↔ 실데이터 정합성 의문** — 전세가율 −23%(보증금이 매매가의 약 23% 수준 → 매우 안전한 구간)인데 결과는 "검토 필요 60점". 원인은 conservative floor(등기 미확인·표본 부족 페널티)이지만, 사용자 입장에서 "안전 구간인데 왜 검토 필요?"라는 인지 부조화가 발생.
2. **핵심 위험 신호가 1건만 보임** — "네이버 검색 기본 부동산 등록 정보 확인" 한 줄. 실거래가/전세가율/표본 부족이 별도 신호로 분해되지 않음.
3. **표본 14건의 출처·시점·단지 매칭 정보 약함** — 어느 단지·어느 기간 표본인지 카드 헤더에 prominent하게 노출되지 않음.
4. **등기부 열람 CTA가 약함** — "수동 분석" 라벨만 보이고, 비용/소요시간/필요 입력값/지금 실행 버튼이 약함.
5. **외부 검색 맥락의 결과 노출** — Search Context Agent 검색 결과가 본문에 어떻게 반영됐는지 불분명. 상태 ✓로만 보이고 본문 카드 매핑이 약함.
6. **다음 확인 액션 우선순위 부재** — 액션이 다수인데 가장 중요한 것 강조 없음.
7. **신뢰구간/표본 분산 미표시** — 평균값만 있고 14건의 분포가 안 보임. 평균이 얼마나 신뢰할 만한지 판단 어려움.

## 추가가 필요한 점

1. **점수 분해 시각화** — `시세 위험 X점 + 권리 미확인 Y점 + 표본 품질 Z점 = 60점`처럼 누적 막대로 분해. "안전한데 검토 필요"의 인지 부조화 해소.
2. **위험 신호 카테고리 그룹화** — `시세 신호` / `권리 신호` / `데이터 품질 신호` 탭/그룹 분리.
3. **등기부 열람 인라인 CTA 카드** — "지금 열람 (수수료 발생, 약 N분)" 버튼 + 필요한 입력값(전화번호·암호·인증번호) 모달.
4. **시세 분포 미니 차트** — sparkline/박스플롯으로 14건 분포 표시. 단지 매칭 N건 / 지역 참고 M건 시각 구분.
5. **단지 매칭 메타데이터 prominent화** — "입력 주소 → 매칭된 단지명 / 매칭 신뢰도 / 최근 거래 기간" 한 줄.
6. **보증보험 가능 여부 자동 판정** — SGI/HUG/HF 기준 룰베이스로 1차 판정. "가입 가능성: 가능 / 조건부 / 불가" 카드.
7. **분석 결과 영구 링크 / PDF 내보내기** — 공인중개사·법률 전문가 공유용.
8. **재분석 diff** — 같은 주소 재분석 시 이전 분석과 차이 비교.
9. **신뢰도 배지** — 각 카드에 "high / medium / low" 표시. 표본 수·데이터 신선도 기반.
10. **모바일 레이아웃 검증** — 데스크탑 정보 밀도가 높아 모바일에서 cognitive load 우려.
11. **에러 케이스 친화적 메시지** — 외부 API 실패 시 카드별 "왜 실패했고 사용자가 무엇을 할 수 있는지" 가이드. 현재 detail은 기술 친화적 메시지.
12. **민감 입력 보호 안내** — 등기부 열람 입력 단계에서 "이 데이터는 서버에 저장되지 않고 CODEF에만 전달됨" 같은 명시.

## 작업 우선순위 (시연/평가 관점)

1. **점수 분해 시각화** — 인지 부조화 즉시 해소, 코드 변경량 적음 → 1순위
2. **위험 신호 카테고리 그룹화** — 1번과 함께 갈 때 시너지 → 2순위
3. **등기부 열람 CTA 강화** — 핵심 차별 기능 노출 강화 → 3순위
4. **시세 분포 미니 차트** — 표본 신뢰도 시각화 → 4순위
5. **보증보험 자동 판정** — 사용자가 가장 궁금해하는 정보 → 5순위
