# 터무니 (Tumuni) — 브랜드 정체성 + 대시보드 재설계

**작성일**: 2026-06-04
**전제**: 트러스트 아크의 부동산 검토 코파일럿을 **위치·업종·결정 통합 코파일럿**으로 확장하며 브랜드 재정립

---

## 1. 브랜드 정체성

### 1.1 이름의 의미

**터무니** = **터**(터전·자리·땅) + **무니**(근거의 옛 표현)

한국어 "터무니없다"는 *근거 없다*는 뜻. 이를 긍정으로 뒤집어 **터무니있는 결정 = 데이터 근거가 있는 위치 기반 결정**으로 재해석.

| 음절 | 의미 | 서비스에서 |
|---|---|---|
| **터** | 부동산의 땅, 창업의 자리, 모든 결정이 이루어지는 공간 | 핵심 데이터 자산 (좌표·건축물·실거래·인허가) |
| **무니** | 근거·이치 | 데이터 출처 명시·점수 분해·agent trace·검증 가능성 |

### 1.2 핵심 정체성 3축

| 축 | 의미 | 트러스트 아크 자산 매핑 |
|---|---|---|
| **터 (Place)** | 위치를 알면 결정이 단단해진다 | 좌표·법정동·건축물대장·실거래 |
| **무니 (Evidence)** | 근거 없는 결정은 터무니없다 | RAG·점수분해·data_statuses·agent trace |
| **결정 (Decision)** | 데이터를 종합해 사용자별 답을 만든다 | Planner·Summarizer·Validation |

### 1.3 브랜드 톤

| 톤 항목 | 가이드 |
|---|---|
| **목소리** | 친근한 전문가 (당근·토스 톤). 어려운 단어 최소화. 단정 회피. |
| **단정 금지** | "안전합니다" "추천합니다" 금지. "현재 데이터 기준 ~", "추가 확인 필요" 유지. |
| **솔직함** | 데이터가 없을 때 "없음" 명시. mock 누수 차단 원칙 그대로. |
| **재해석 유머** | "터무니없는 결정 대신" 같은 부드러운 위트 허용 (가벼움 ≠ 단정) |

## 2. 슬로건

### 2.1 메인 슬로건 (정착 안)

> **"터무니있는 결정의 시작"**

- 짧다 (8음절)
- 한국어 임팩트 강함 ("터무니없는"의 긍정 재해석)
- 서비스 본질 (근거 기반 의사결정) 즉시 전달
- 활용도: 헤더, 메타 description, 발표 자료, 로딩 화면

### 2.2 서브 슬로건 / 변형 5종

| 용도 | 카피 |
|---|---|
| 첫 화면 헤드라인 | **"터를 읽고, 무니를 더하다."** |
| 결과 화면 안내 | **"당신의 자리, 데이터로 터무니있게."** |
| 발표·보도자료 | **"위치를 알면, 결정이 단단해진다."** |
| 부동산 모드 | **"임차·매수, 터무니있는 검토부터."** |
| 창업 모드 | **"이 자리에서, 터무니있는 시작."** |
| 광고·외부 | **"근거 없는 결정 말고, 터무니있는 결정."** |

### 2.3 마이크로카피 톤 예시

## 3. 모드 프레임워크

기존 부동산 검토에서 **3개 모드**로 확장. 같은 11개 코어 agent 인프라를 모드별로 다르게 활용.

### 3.1 모드 ① — 부동산 검토 (기존 강화)

```
"이 매물 임차·매수, 터무니있게 검토합니다."
```

- 대상: 임차인, 매수자
- 기존 분석 흐름 유지 + 점수 분해 + Planner intent
- 추가될 기능: 매수vs전세 시뮬, 호가 적정성, 임대수익률 (앞서 추천한 5개)

### 3.2 모드 ② — 창업 인허가 (신규)

```
"이 자리에서 시작 가능한지, 터무니있게 알려드립니다."
```

- 대상: 예비 창업자, 자영업자, 임대인
- 핵심: 위치 + 업종 → 인허가 가능 여부·필요 절차·예상 비용
- 신규 agent 6개 (Zoning · Building Suitability · School Zone · Youth Protection · Competition Density · License Requirement)

### 3.3 모드 ③ — 상가 활용성 (통합)

```
"빈 상가, 어떤 자리로 살릴 수 있는지 터무니있게 매칭합니다."
```

- 대상: 상가 임대인, 매수자, 공인중개사
- 핵심: 부동산 분석 + 가능 업종 매칭 + 임대수익률 시뮬
- 디엘톤 발표 시연용 핵심 기능

### 3.4 모드 전환 UI

대시보드 상단에 탭 형태로 배치:

```
[ 부동산 검토 ] [ 창업 인허가 ] [ 상가 활용성 ]
```

선택한 모드에 따라 입력 폼·결과 섹션 일부가 달라짐. 공통 인프라(좌표·건축물·점수분해)는 동일.

## 5. 컬러·타이포 (현재 팔레트 강화)

기존 트러스트 아크의 moss·cream·ink·brass·clay 팔레트는 **이미 한국적 따뜻함**이 있어 터무니 브랜드에 잘 맞습니다. 그대로 유지하되 의미를 재정의합니다.

| 컬러 | 헥스 추정 | 기존 의미 | 터무니 의미 |
|---|---|---|---|
| **moss** | 짙은 초록 | 안전·성공 | **무니 (근거 충실)** |
| **brass** | 따뜻한 황금 | 중간·검토 | **터 (땅·자리)** |
| **clay** | 적갈색 | 위험·미확인 | **근거 부족 / 한계** |
| **ink** | 진한 먹색 | 본문 | 본문 (그대로) |
| **cream** | 따뜻한 백 | 배경 | 배경 (그대로) |

**핵심 점수 카드의 색상 규칙**:
- 터무니 80+ : moss (근거 충실)
- 터무니 60~79 : brass (보강 필요)
- 터무니 60 미만 : clay (근거 매우 부족)

타이포는 현재 serif/sans 조합 유지. 한국어 본문은 한국어 sans (Pretendard, Noto Sans KR 권장). 헤드라인은 현재 serif 유지(친근감).

### 5.1 로고 컨셉 제안

세 가지 방향. 디자이너 의뢰 시 참고용.

**A. 한자 + 무니** : "**터**" 한자 위에 작은 도트(데이터 점) 더하기
**B. 도장 (印) 모티프** : 인장 모양 안에 "터무니" 한글. 근거를 인증하는 느낌
**C. 매듭 (지문 매듭) 모티프** : 한국 매듭의 단순화된 형태 + 점 (데이터)

추천: **B. 도장** — "근거를 찍어 인증한다"는 의미가 서비스 본질과 가장 잘 맞음.

## 7. 모드별 입력 폼 / 결과 차이

### 7.1 부동산 검토 모드 (기본)
입력: 계약 유형 · 주택 유형 · 주소 · 보증금/월세/매매가 · 자유 질문
결과: 기존 화면 + 새 섹션명 적용

### 7.2 창업 인허가 모드 (신규)
입력: **업종** · **점포 주소** · **점포 면적** · **운영 시간** · **자유 질문**
결과 차이:
- 종합 위험도 → **터무니지수 (영업 가능성)**
- 실거래 근거 → **상권 근거** (동종업종 밀집도)
- 권리 근거 → **인허가 근거** (필요 절차·수수료)
- 건축 근거 → **시설 적합성 근거** (용도지역+건축물대장)
- 신규 섹션: **영업제한 거리 검증** (학교·청소년시설·동종업종)

### 7.3 상가 활용성 모드 (디엘톤 demo 핵심)
입력: **상가 주소** · **목적** (임대/매수/창업) · **자유 질문**
결과:
- 부동산 가치 + 가능 업종 매칭 + 임대 수익률 시뮬
- 한 화면에 부동산·창업·시장 데이터 결합
- 디엘톤 시연용 시그니처 기능
