## 1. API 신청 — 사용자(귀하)가 할 일

기존 보유: `data.go.kr` 키 (실거래가용)
**오늘 일괄 활용 신청** (모두 무료, 1~3일 자동 승인)

### 1.1 창업 모드 활성화 필수 6개

| # | API | 신청처 | 즉시 발급? | 용도 |
|---|---|---|---|---|
| 1 | **LOCALDATA** (행정안전부) | https://www.localdata.go.kr 회원가입 | ✅ 즉시 | 전국 1,700+ 자치단체 인허가 사업장 전수 (200+ 업종 좌표) — 동종업종 밀집도·이격거리 |
| 2 | **국토부 LURIS 토지이용계획** | data.go.kr 활용신청 | 1~3일 | 용도지역·용도지구 |
| 3 | **법제처 국가법령정보 Open API** | https://www.law.go.kr/DRF/ 본인인증 | ✅ 즉시 | 식품위생법·학원법·공중위생법·게임산업법 본문 |
| 4 | **자치법규정보시스템(ELIS)** | data.go.kr 활용신청 | 1~3일 | 자치구별 영업제한 거리·시간 조례 |
| 5 | **학교알리미 학교 기본정보** | data.go.kr 활용신청 | 1~2일 | 정화구역 200m 검증용 학교 좌표 |
| 6 | **여성가족부 청소년시설** | data.go.kr 활용신청 | 1~2일 | 청소년 보호 거리 검증 |

### 1.2 상가 활용성 추가 2개 (1~6번과 거의 모두 공유)

| # | API | 신청처 | 즉시 발급? | 용도 |

### 1.3 신청 시 활용 사례 (복사용)
> "위치 기반 의사결정 코파일럿 서비스 개발용. 사용자가 입력한 점포 주소·업종을 받아 토지이용계획·법령·인허가 사업장·조례 데이터를 결합해 인허가 가능 여부와 절차를 안내하는 정보 제공형 분석에 활용. 상업 이용 가능 범위 내에서 가공·재배포 없이 분석 결과만 사용자에게 노출."

## 2. 데이터 수집·전처리 (자동화 작업)

### 2.1 한 번만 하면 되는 작업

| 작업 | 산출물 | 추정 시간 |
|---|---|---|
| 법제처에서 4개 법령 다운로드 + 시행령·시행규칙 | `rag_docs/laws/*.md` (4~6 파일) | 반나절 |
| 서울 25개 자치구 조례 다운로드 (ELIS) → RAG 인덱싱 | `rag_docs/ordinances/*.md` (25 파일) | 1일 |
| 업종별 시설기준 매핑 테이블 (수기 작업) | `lib/server/rules/business-facility.json` | 1~2일 |
| 행정동코드 ↔ 관할 보건소/구청 매핑 | `lib/server/rules/jurisdiction.json` | 반나절 |
| LOCALDATA 일별 dump 다운로드 + 캐싱 전략 | Vercel KV 또는 in-memory cache | 1일 |

### 2.2 정기 갱신
- 법령·조례 RAG 재인덱싱 (개정 시 — 월 1회 cron 권장)
- LOCALDATA 일별 갱신 (자동 — 신규 개·폐업 데이터 포함)

### 2.3 RAG 인덱싱 방식 권장
- 임베딩: OpenAI `text-embedding-3-small` (저렴, 충분한 품질)
- 벡터 저장: Vercel KV 또는 Postgres + pgvector
- 청킹: 법령은 조문 단위(article), 조례는 항·호 단위
- 메타데이터: `{law_name, article_no, last_updated, jurisdiction}`
- MVP에서는 in-file JSON으로 시작 → 필요 시 벡터 DB로

## 3. 코드 작업 — 새 에이전트 작성

### 3.1 창업 모드 6개 에이전트

| Agent | 핵심 API | 산출 데이터 | 추정 시간 |
|---|---|---|---|
| **Zoning Agent** | LURIS | 용도지역 명·가능 업종 리스트·불가 사유 | 1일 |
| **Building Suitability Agent** | 건축물대장 + 업종 룰 | 주용도 매칭·시설 기준 적합 여부 | 1일 |
| **School Zone Agent** | 학교알리미 + 반경 계산 | 200m 내 학교 N개·정화구역 판정 | 1일 |
| **Youth Protection Agent** | 청소년시설 + 반경 | 청소년 보호 거리 위반 여부 | 0.5일 |
| **Competition Density Agent** | LOCALDATA + 반경 500m | 동종업종 N개·1년 신규·1년 폐업 | 1일 |
| **License Requirement Agent** | 법령 RAG + 조례 RAG + 룰 | 신고 필요 여부·예상 수수료·처리 기간·결격 사유 | 1.5일 |

각 에이전트는 이미 작성된 `lib/server/data-go-kr.ts` 헬퍼 위에서 30~80줄 wrapper만 추가하면 됨.

### 3.2 상가 활용성 4개 에이전트

| Agent | 데이터 | 산출 데이터 | 추정 시간 |
|---|---|---|---|
| **Property Value Agent** | 부동산 모드 분석 재사용 | 실거래 평균·시세 적정성 | 0.5일 |
| **Possible Business Agent** | LURIS + 건축물대장 + 업종 룰 | 가능 업종 TOP 5 + 각 적합도 | 1일 |
| **Cap Rate Agent** | 임대료 통계 + 매매가 | 업종별 임대수익률 시뮬레이션 | 0.5일 |
| **Vacancy Trend Agent** | 소상공인진흥공단 상권 | 신규 진입·폐업 패턴·상권 활성도 | 0.5일 |

### 3.3 route.ts 통합

현재 placeholder 분기를 실제 호출로 교체:

```ts
if (payload.mode === "business_permit") {
  const zoning = await runZoningAgent(...);
  const building = await runBuildingSuitabilityAgent(...);
  const schoolZone = await runSchoolZoneAgent(...);
  // ... 나머지 3개
  const scored = runBusinessRiskScoringAgent({ ... });
  const summary = await runSummarizerAgent({ ... }); // 기존 LLM 재사용
  return NextResponse.json(...);
}
```

### 3.4 Planner 시스템 프롬프트 모드별 확장
- 부동산 모드: 현재 그대로
- 창업 모드: intent_tags에 `concern_zoning`·`concern_school_zone`·`concern_competition` 추가 / execution_plan에서 critical 단계가 업종별로 다름 (PC방 = School Zone critical, 음식점 = Building+Zoning critical)
- 상가 활용성: `concern_lease_yield`·`concern_business_match` 추가

## 9. 사용자가 API 신청하는 동안 제가 할 수 있는 작업

승인 대기 1~3일 동안 미리 준비 가능한 것들:

### 9.1 placeholder 디테일 정비 (오늘 가능, 2~3시간)
- `confirmed_facts` 채우기 (입력값 echo)
- 모드별 빈 상태 카피 정비
- 점수 카드 기본 색상 가독성 점검

### 9.2 LOCALDATA wrapper 함수 초안 (응답 스키마 미리 추정, 3~4시간)
- 공식 문서에서 응답 스키마 추측 → `lib/server/localdata.ts` 작성
- 좌표 반경 필터 함수
- 동종업종 카운트 헬퍼

### 9.3 식품위생법 RAG 인덱싱 파이프라인 (반나절)
- 법제처에서 식품위생법 본문 다운로드
- 조문 단위 chunking 함수
- 간단한 in-file JSON 저장 (벡터 DB는 추후)

### 9.4 시설기준 룰 테이블 음식점 부분 (반나절)
- 식품위생법 시행규칙 별표 1~3 기준 정리
- JSON 룰로 변환

위 4가지를 신청 대기 중 작업하면 키 받자마자 즉시 활성화 가능.

| **전체 작업** | 3~3.5주 (표준), 5일 (디엘톤 음식점 1개 단축) |
| **새 에이전트** | 창업 6개 + 상가 4개 = 10개 |
| **재사용** | 부동산 모드 인프라 (좌표·건축물·실거래) + Planner·Summarizer·점수분해·Trace |
| **첫 액션 (사용자)** | data.go.kr에서 6개 API 일괄 활용 신청 + LOCALDATA 회원가입 |
| **첫 액션 (개발)** | placeholder 디테일 정비 + LOCALDATA wrapper 초안 + 식품위생법 RAG |
| **디엘톤 단축 경로** | 음식점 1개 업종 + 2개 API (LOCALDATA + LURIS)로 1주 안에 라이브 |
